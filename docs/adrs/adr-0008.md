# ADR-0008: Reject BMI2 Quote-Indexing for YAML (P6)

## Status

✅ Accepted

## Context

Succinctly's DSV parser uses BMI2 bit-manipulation (`PDEP`/`PEXT`, and a `toggle64` trick) to
build a **global quote index** for a document in a single pass: because CSV quoting is
context-free (a doubled `""` is the only escape), the parser can compute, for all bytes at once,
which are inside quotes. Optimization **P6** proposed bringing the same BMI2 quote-indexing
technique to the YAML oracle (ADR-0003).

## Decision

We will **not** use DSV-style BMI2 quote-indexing in the YAML parser.

The blocker is grammatical, not a matter of tuning: **YAML's escaping is not context-free**, so
the one-pass global quote index that works for CSV cannot be built for YAML.

- CSV escapes with `""`, which is context-free — a quote mask can be computed directly with BMI2.
- YAML escapes with backslashes (`\"`), so detecting real quote boundaries requires first
  computing an **escape mask** — but escape handling is exactly what the optimization was meant to
  accelerate. This is a circular dependency: you need the escape mask to build the quote mask, and
  the quote mask to handle escapes.
- Resolving it needs a multi-pass approach (escape → quote → parse) that is **slower** than the
  current single-pass early-exit SIMD.

YAML adds further context-sensitivity that CSV lacks: two quoting styles (`"` vs `'`) with
different escaping rules, block scalars (`|`, `>`), and quote characters whose meaning depends on
context. The original micro-benchmark also compared the wrong thing — per-string parsing (where
early-exit SIMD wins on short strings) rather than global index building — which masked the
fundamental grammar difference. Rejected 2026-01-17.

Full analysis: [parsing/yaml.md § P6](../parsing/yaml.md#p6-bmi2-operations-pdeppext---rejected-).

## Consequences

- YAML string scanning keeps its single-pass early-exit SIMD approach; BMI2 quote-indexing
  remains a DSV-only technique.
- Records a reusable boundary: **BMI2 global quote-indexing applies only when escaping is
  context-free (CSV's `""`), not to grammars with backslash escaping or context-dependent quoting
  (YAML)**. This prevents re-attempting the port.
- Neutral: the difference is intrinsic to the formats, so the decision is unlikely to change
  unless YAML's grammar constraints do.
