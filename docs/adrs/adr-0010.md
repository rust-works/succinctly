# ADR-0010: Reject AVX-512 SIMD Variants (P8)

## Status

✅ Accepted

## Context

The YAML oracle's SIMD routines (ADR-0003) use AVX2 (256-bit) on x86_64. Optimization **P8**
proposed adding AVX-512 (512-bit) variants to process twice as many bytes per instruction.
Micro-benchmarks appeared to show a 2× speedup at 256 bytes.

There was also direct precedent: an AVX-512 path had already been tried for **JSON** and found to
be 7–17% *slower* than AVX2, and was removed from the codebase
([optimizations/history.md](../optimizations/history.md)).

## Decision

We will **not** add AVX-512 variants to the YAML parser.

Two independent reasons:

1. **Memory-bound workload.** YAML parsing is sequential text scanning, bottlenecked on memory
   bandwidth. AVX2 already saturates available bandwidth, so wider vectors cannot help — the same
   reason the JSON AVX-512 path was removed. At realistic sizes on a Ryzen 9 7950X: 64 B was **7%
   slower** (18.59 ns vs 17.34 ns), 128 B was break-even, and apparent wins only appeared at 256 B+.
2. **Benchmark design flaw.** The micro-benchmark measured *loop iterations*, not work performed.
   For 256 bytes AVX2 runs 8 iterations (32 B chunks) and AVX-512 runs 4 (64 B chunks), so the
   "2× speedup" was simply half the iteration count, not twice the per-byte throughput.

On Zen 4 there is a further penalty: 512-bit ops are split into two 256-bit micro-ops, adding
overhead without benefit. Rejected 2026-01-18.

Full analysis: [parsing/yaml.md § P8](../parsing/yaml.md#p8-avx-512-variants---rejected-).

## Consequences

- The parser targets AVX2 as the widest x86_64 vector width; NEON on ARM64. No AVX-512 path is
  maintained.
- Confirms a codified rule for this project: **wider SIMD ≠ faster for memory-bound workloads** —
  once AVX2 saturates RAM bandwidth, AVX-512 adds cost, not throughput. It also flags a benchmark
  anti-pattern: **measure work performed (bytes/time), not loop iterations.**
- Together with ADR-0007 (size mismatch), ADR-0008 (grammar mismatch), and ADR-0009 (use-case
  mismatch), this completes a family of SIMD/optimization rejections, each stemming from a
  different mismatch between a micro-benchmark and reality.
